Arquitetura & Conformidade

Perguntas Frequentes

Respostas diretas sobre a engenharia, segurança e operação do sistema.

O sistema permite a exportação de dados? Como é feito o processo?

Sim, totalmente. A plataforma possui um **motor de extração nativo** integrado diretamente na interface de listagem de ativos. O processo é instantâneo: basta aplicar os filtros desejados (como filial, departamento ou período) e clicar no formato pretendido para descarregar os dados processados em tempo real, sem intermediários.

Quais formatos de ficheiro são suportados na exportação de ativos?

Para garantir total interoperabilidade, o sistema suporta 5 formatos estratégicos: **Excel (.xlsx)** para gestão financeira clássica; **CSV** otimizado para ferramentas de BI e Data Science; **PDF** com assinaturas criptográficas pronto para auditorias oficiais; além de **JSON** e **XML** para integração direta com ERPs modernos ou sistemas legados (como SAP e Primavera).

Qual é a vantagem estratégica deste modelo de extração para a minha empresa?

A principal vantagem é a **eliminação do aprisionamento tecnológico (vendor lock-in)**. Os dados patrimoniais pertencem à sua empresa. Ao expor os dados em formatos universais e estruturados, o seu negócio ganha autonomia para migrar, cruzar informações em ecossistemas de Business Intelligence ou alimentar auditorias externas sem custos adicionais de desenvolvimento.

Qualquer utilizador pode exportar a base de dados de ativos completa?

Não. A extração de dados em lote é protegida pelo sistema de controlo de acesso baseado em funções (RBAC). É possível bloquear ou permitir a exportação de relatórios sensíveis de forma cirúrgica, garantindo que apenas perfis autorizados (como Diretores Financeiros ou Auditores Chefes) tenham acesso aos downloads em massa.

Como garantir que os dados exportados correspondem à realidade do inventário?

Cada alteração no sistema gera um Hash SHA-256 encadeado. Ao exportar relatórios oficiais (como o formato PDF), o sistema injeta a assinatura criptográfica correspondente do estado atual do banco de dados. Qualquer tentativa de adulteração manual nos dados após a exportação poderá ser facilmente detetada cruzando a assinatura com o servidor local.
